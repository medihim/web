광고와 매출 데이터를 연결하는 최소 구조

복잡한 시스템보다 공통 식별자와 단계 정의가 먼저입니다. 광고에서 실제 매출까지 연결하는 최소 데이터 구조입니다.

핵심 결론

광고와 매출 연결의 출발점은 고가 솔루션이 아니라, 유입 식별자·상담 ID·상태 정의를 일관되게 기록하는 것입니다.

1. 작동 원리

1. 한 리드에 하나의 식별자를 부여합니다

채널마다 이름과 전화번호를 다르게 저장하면 중복과 성과 귀속 오류가 생깁니다.

2. 원천값을 보존합니다

최초 UTM과 캠페인·소재 정보를 덮어쓰지 않고 후속 접점과 분리해 저장합니다.

3. 단계 정의를 통일합니다

문의, 유효상담, 예약, 내원, 매출의 조건을 팀 전체가 동일하게 사용해야 합니다.

2. 실행 구조

광고UTM·캠페인·소재 ID
랜딩세션·폼·채널 클릭
상담상담 ID·국가·시술·상태
예약예약일·보증금·변경
매출내원·수납·환불·공헌이익

3. 핵심 지표

소스 식별률신규 리드 중 출처가 확인된 비율
단계 완결률필수 상태와 날짜가 모두 기록된 비율
매출 연결률내원 매출 중 상담 ID가 연결된 비율
중복률동일 환자가 여러 리드로 생성된 비율

4. 실행 체크리스트

목표와 단계의 정의가 팀 전체에 동일한가
담당자·기한·기록 위치가 지정되어 있는가
광고부터 상담·예약·매출까지 식별자가 연결되는가
주간 단위로 원인과 다음 실험을 결정하는가

5. 리스크 및 검수 포인트

진료정보와 마케팅 데이터를 목적 없이 합치지 않습니다. 최소수집, 접근권한, 보관기간, 파기, 처리위탁, 국외 이전과 동의 요건을 설계 단계에서 검토합니다.

본 콘텐츠는 병원 사업·운영을 위한 일반 정보이며, 의료·법률·세무 자문을 대체하지 않습니다. 실제 적용 시 관련 전문가와 최신 법령을 확인해야 합니다.
